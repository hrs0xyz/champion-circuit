  On load:
    → Default: "All Cities" selected, "All sports" selected
    → Auto-selects user's city from profile (once per session,
      only if no city was previously chosen)

  City Dropdown (top-left):
    → Click to open panel (220px wide)
    → Search box to filter cities as you type
    → Options: All Cities + Kolkata, Mumbai, Delhi, Bengaluru,
               Hyderabad, Chennai, Pune, Ahmedabad
    → Selecting a city filters venues to that city
    → Selecting "All Cities" clears filter (persists for session)
    → City selections are tracked in user activity log

  Sport Filter chips (scrollable row):
    All sports | BGMI | Badminton | Basketball | Boxing/MMA |
    Call of Duty | Carrom | Chess | Cricket | EA FC/FIFA |
    Football | Free Fire | Kabaddi | PC Gaming | PUBG |
    PlayStation | Swimming | Table Tennis | Tennis | Valorant | Volleyball

    → Selecting a sport:
        Filters venues to only those offering that sport
        Shows sport-specific cover photo on venue cards
        Renames venue card button to "View Badminton →" etc.
        Tracked in user activity log

  Venue Card:
    → Sport-specific photo (Unsplash fallback if no cover)
    → Sport badge (BADMINTON / CRICKET / etc.) when filtered
    → Verified badge
    → Venue name, city/state
    → Matching listing name (e.g. "Badminton Court 1")
    → Description
    → 📞 Call button | 🗺 Maps button
    → [ View Badminton → ] / [ View listings → ]
        Navigates to /venue/:id?sport=badminton

  Activity logged:
    → Sport filter click → event: sport_filter
    → City selection    → event: city_filter
    → Venue card click  → event: venue_card_click

  USER ACTIVITY LOG (automatic, fire-and-forget)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Logged for authenticated users only.
  Stored in: SQLite → user_activity table
  Never blocks UI (async, silent failures ignored)

  Events tracked:
    venue_view       → User opens a venue detail page
    venue_card_click → User clicks a venue card on /turf
    listing_inquiry  → User clicks Send Inquiry on a listing
    sport_filter     → User selects a sport chip on /turf
    city_filter      → User selects a city from dropdown

  Each event captures:
    user_id, username, event, venue_id, venue_name,
    sport, city, listing_id, listing_title, timestamp

  Access:
    Admin: /staff/admin → "📊 Activity" tab
      → Insights view: top venues, top sports, top cities,
        views-to-inquiry funnel (flags low-converting venues ⚠)
      → Full log: filterable table with CSV export
    Venue owner: /staff/admin → Activity tab (own venue only)
